# लॉगिन मॉड्यूल (Login Module) - तांत्रिक माहिती व सीनेरिओज

हा डॉक्युमेंट 'Hi-Tech Dairy POS' सिस्टीमच्या 'लॉगिन' प्रक्रियेबद्दल (Login Flow) तांत्रिक आणि व्यावहारिक माहिती देतो. 

---

## १. सुरवात (Health Check)
जेव्हा तुम्ही लॉगिन स्क्रीन उघडता, तेव्हा सिस्टीम सर्वात आधी बॅकएंड सर्व्हर सुरू (Online) आहे की नाही, हे तपासते.
- **API Call:** `GET api/v1/auth/health`
- **उपयोग:** जर सर्व्हर बंद असेल किंवा इंटरनेट नसेल, तर सिस्टीम लगेच "Offline" असल्याचा मेसेज दाखवू शकते.

## २. सुरक्षा आणि एन्क्रिप्शन (RSA Handshake)
लॉगिन करताना तुमचा युझरनेम (Username) आणि पासवर्ड (Password) थेट (Plain Text मध्ये) पाठवला जात नाही. तो अत्यंत सुरक्षित पद्धतीने पाठवला जातो.
- **Handshake API:** `POST api/v1/auth/handshaking`
- **प्रक्रिया:** सुरुवातीला फ्रंटएंड एक 'Public Key' बॅकएंडला पाठवतो. त्यावरून फ्रंटएंड आणि बॅकएंड यांच्यात एक सुरक्षित कनेक्शन तयार होतं. 
- **AES Encryption:** तुम्ही टाकलेला युझरनेम आणि पासवर्ड हा `rsaService.aesEncrypt()` द्वारे एन्क्रिप्ट (लपवला) जातो. 

## ३. लॉगिन करणे (Login API)
सर्व माहिती सुरक्षित झाल्यावर प्रत्यक्ष लॉगिनचा कॉल जातो.
- **API Call:** `POST api/v1/auth/login`
- **Payload:** यामध्ये एन्क्रिप्ट केलेला डेटा पाठवला जातो.
- **यशोगाथा (Success):** जर लॉगिन यशस्वी झालं, तर बॅकएंड आपल्याला युझरची माहिती (उदा. User ID, Role, Organization ID) आणि 'Tokens' परत देतं.

## ४. डेटा सेव्ह करणे (Local Storage)
लॉगिन यशस्वी झाल्यावर मिळालेली माहिती ब्राउझरच्या **Local Storage** मध्ये सेव्ह केली जाते.
- **सेव्ह होणारा डेटा:** `UserDetails` (युझरची माहिती), `Token` (पुढच्या API कॉल्ससाठी सिक्युरिटी पास).
- **Remember Me:** जर तुम्ही 'Remember Me' वर टिक केली असेल, तर तुमचा युझरनेम तिथे सेव्ह राहतो, जेणेकरून पुढच्या वेळी पुन्हा टाईप करावा लागू नये.

## ५. रीडायरेक्ट (Redirection)
डेटा सेव्ह झाल्यानंतर:
- तुम्हाला आपोआप **POS (Counter Sale)** किंवा **Dashboard** स्क्रीनवर पाठवले जाते (Redirect केले जाते). 
- आणि त्यानंतर वर सांगितल्याप्रमाणे 'Master Data' (प्रॉडक्ट्स, ग्राहक इ.) लोड व्हायला सुरुवात होते.
