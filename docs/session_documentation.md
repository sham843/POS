# सेशन (Day/Shift Session) - तांत्रिक माहिती व सीनेरिओज

हा डॉक्युमेंट 'Hi-Tech Dairy POS' मधील **Session Start** (दिवस/शिफ्ट सुरू करणे) आणि **Session End** (गल्ला बंद करणे) या प्रक्रियेबद्दल सविस्तर माहिती देतो. 

---

## १. सेशन सुरू करणे (Session Start)
जेव्हा तुम्ही पहिल्यांदा POS सुरू करता, तेव्हा तुम्हाला तुमची शिफ्ट किंवा गल्ला सुरू करण्यासाठी 'Start Session' करावे लागते.
- **API Call:** `POST api/v1/session/start`
- **पाठवला जाणारा डेटा (Payload):**
  - `UserId` (तुमचा आयडी)
  - `OpeningBalance` (सकाळी गल्ल्यात असलेली रोख रक्कम)
  - `ShiftId` आणि `StartTime`
- **प्रक्रिया:** 
  - बॅकएंडमधून एक नवीन **Session ID** तयार होतो. 
  - हा Session ID तुमच्या ब्राउझरमध्ये (Local Storage) सेव्ह केला जातो.
  - यापुढील दिवसभरातील सर्व बिलांवर (Invoices) हाच Session ID लावला जातो, जेणेकरून त्या शिफ्टमध्ये किती गल्ला झाला हे ओळखता येतं.

## २. सेशन दरम्यान (During the Session)
- प्रत्येक बिलावर तुमचा Session ID सेव्ह होत असल्यामुळे, तुम्ही दिवसभरात किती रोख (Cash) आणि किती ऑनलाईन (Online) पैसे कमवले, याचा अचूक हिशोब बॅकएंडला मिळतो.
- जर चुकून कॉम्प्युटर बंद पडला, तरी Session ID सेव्ह असल्यामुळे तुम्ही पुन्हा लॉग-इन करून त्याच सेशनमध्ये काम सुरू ठेवू शकता.

## ३. सेशन संपवणे (Session End / Day Close)
जेव्हा तुमची शिफ्ट किंवा दिवसाचं काम संपतं, तेव्हा गल्ला मोजून सेशन बंद करावं लागतं.
- **Summary API Call:** `GET api/v1/session/today-summery/{userId}`
  - हा API दिवसभराचा पूर्ण हिशोब (सकाळची रक्कम, कॅश कलेक्शन, ऑनलाईन कलेक्शन, खर्च) घेऊन येतो आणि **Expected Closing Balance** (अपेक्षित गल्ला) दाखवतो.

- **गल्ला मोजणे (Physical Count):** 
  - सिस्टीममधील गल्ला आणि प्रत्यक्षात हातातील गल्ला यात काही फरक (Difference) आहे का, हे पाहण्यासाठी तुम्हाला 'Actual Balance' (हातात असलेली रक्कम) टाकावी लागते.
  - कारणे किंवा Remarks देण्याचीही सुविधा असते.

- **सेशन फायनल करणे (Confirm End Session):**
  जेव्हा तुम्ही 'End Session' बटण दाबता, तेव्हा दोन APIs लागोपाठ कॉल होतात:
  १. **`POST api/v1/session/save-summary`**: हा API दिवसभराचा संपूर्ण हिशोब आणि तुम्ही टाकलेली माहिती कायमस्वरूपी सेव्ह करतो.
  २. **`POST api/v1/session/end`**: हा API तुमचं सेशन अधिकृतरित्या 'Closed' मार्क करतो, सोबत Closing Balance आणि Difference पाठवतो.

## ४. लॉग-आऊट (Logout)
- सेशन यशस्वीरित्या बंद झाल्यावर सिस्टीम आपोआप तुमची सर्व कॅश मेमरी (Local & Session Storage) पुसून टाकते आणि तुम्हाला लॉग-आऊट करून बाहेर काढते.
- दुसऱ्या दिवशी तुम्हाला पुन्हा नवीन 'Start Session' करून काम सुरू करावे लागते.
